CloudFormation

CloudFormation key points:

  • Used to provision AWS resources with "Infrastructure as Code" (think Terraform).
  • Write templates in either JSON or YAML formats (preferred by most).
  • Stacks refer to a group of resources created by a particular template.
  • Templates larger than 51,200 bytes (0.05 MB) are too large to upload directly, and must be imported into CloudFormation via an S3 bucket.
  • NestedStacks allow users to create smaller reusable templates and use them together in a larger template.
  • Stack deployment error will show the message "ROLLBACK_IN_PROGRESS".
  • Templates need to have at least one Resource defined under the Resource section in order to be valid.

CloudFormation template sections:

  • MetaData: extra information about the template
  • Description: a description of the template's purpose
  • Parameters: how to implement user input into a template
  • Transforms: applies macros
  • Outputs: values used to be imported into other stacks
  • Mappings: maps keys to values
  • Resources: defines resources to be provisioned, at least one resource is mandatory
  • Conditions: conditional expression for whether resources or properties are created
