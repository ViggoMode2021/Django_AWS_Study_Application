EC2 (Elastic Compute Cloud)

EC2 key points:

  • Provision instances of virtual servers in the cloud

  • Configure EC2 instance by selecting: OS, Storage, Memory, Network Throughput and Linux is the most popular instance type

  • Select an AMI (Amazon Machine Image) as the specific instance type

  • Instance types will typically double in price and key attributes

  • Placement Groups allow for organization of instances to ensure performance, communication, and reliability

  • Instance profiles are a container for an IAM role to pass IAM credentials to an EC2 instance during startup

  • Security Group is a firewall that allows inbound and outbound rules for instance traffic

  • SSH (Secure Shell) is a networking tool that allows users to enter their instance from the command line. Must need to create a key pair during start up typically in order to access.

  • SSH (Secure Shell) requires port 222 to be open for inbound traffic. This can be configured in the EC2 instance's security group.

  • UserData is a script that will run automatically when creating an instance. Used to configure settings and download packages to ensure customization prior to instance creation without need to SSH

  • MetaData contains metadata about an instance. When SSHd into an instance, curl http://169.254.169.254/latest/meta-data in order to access metadata.

  • MetaData could be instance type, ip addresses etc.
