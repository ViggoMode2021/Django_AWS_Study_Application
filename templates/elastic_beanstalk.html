Elastic Beanstalk

Elastic Beanstalk key points:

  • Handles deployment, load balancing, capacity, auto-scaling, and health monitoring of applications

  • Free to use, only pay for the provisioned infrastructure (EC2, ALB etc.)

  • Supports runtimes: Docker, Python, Go, Ruby, Java, Node.js, .NET, PHP

  • Application projects need a .ebextensions folder placed at the root of the projects. This folder contains necessary configuration files.

  • Two different environment types to launch: Web or Worker Environments

  • Web Environments can be Single-Instance or Load-Balanced

    • Single-Instance Env launches a single EC2 instance, an EIP is assigned to the EC2

    • Load Balanced Env launch EC2s behind an ELB managed by an ASG


  • Worker Environments create an SQS queue, install the SQS daemon on the EC2 instances, and have an ASG policy, which will add or remove instances based on queue size.


  • EB has the following deployment policies:

    • All at once: takes all servers out-of-service, applies changes, puts servers back-in-service. Fast, but has application downtime.

    • Rolling: updates servers in batches, reduced capacity based on batch size. This policy needs an ELB and cannot be done on single instance environments.

    • Rolling with Additional Batch: Adds new servers in batches to replace old ones. This policy needs an ELB and cannot be done on single instance environments. This policy never reduces application capacity.

    • Immutable creates same number of servers, switches traffic to new servers, and then removes old servers.


  • In-Place deployment is when deployment occurs within the environment. All deployment policies are In-Place.


  • Blue/Green is when deployment swaps environments outside an environment. Blue/Green is recommended when there are external resources (i.e. - An RDS db instance)
