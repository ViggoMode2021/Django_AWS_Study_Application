Lambda

Lambda key points:

  • Lambda functions are serverless functions that run in the cloud. Function code runs without provisioning and managing servers.

  • Run tasks that last for up to 15 minutes where the os environment doesn't need customization. Otherwise, use Fargate.

  • There are 7 runtime language environments officially supported by Lambda: Ruby, Python, Go, Node.js, Java, PowerSchool, C#

  • A Cold Start is when a Lambda function has not been recently executed. This means that there will be a delay

  • Can be triggered via from SDK or via services like S3, DynamoDB, API Gateway (that run in no VPC). However, to interact directly with some services, Lambda function needs to be in same VPC as the other service.

  • You can adjust the duration timeout for up to 15 mins and memory up to 3008 MB

  • Lambda can scale up to 1000 concurrent functions in seconds. 1000 is the default, but AWS Service Limit Increase will allow this metric to be raised.

  • Handler is the method in your function code that processes events. When your function is invoked, Lambda runs the handler method. When the handler exits or returns a response, it becomes available to handle another event.
